Encryption Process

  1. Validation: Request text is checked before storage.
  2. AES-256: Case details are encrypted using a unique IV.
  3. TLS Storage: Data is stored over an encrypted database connection.
  4. Access Control: Only assigned lawyers can read the case.
  5. Audit Trail: Access is logged and timestamped.

What Gets Protected

  • Case description content
  • Encrypted storage at rest
  • Random IV for every submission
  • Restricted access by role

Privacy & Confidentiality
